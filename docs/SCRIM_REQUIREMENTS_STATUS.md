# 스크림 관리 시스템 구현 현황

> 2026-07-09 방향 변경: 별도 스크림 운영 기능은 더 이상 확장하지 않는다.
> 스크림 일정 조율은 Discord에서 진행하고, 앱의 기존 스크림 화면은
> 팀 목록과 공식 경기 결과를 다루는 `대회 진행방`으로 전환한다.
> 아래 내용은 기존 구현 기록이며 신규 작업 범위로 보지 않는다.

상태 기준:

- 완료: API와 화면 흐름이 현재 앱에 있음
- 부분: DB 스키마 또는 일부 API는 있으나 관리 화면/수정/삭제 등 운영 기능이 부족함
- 미완료: 아직 사용자 기능으로 연결되지 않음

## 1. 회원 관리

| 항목 | 상태 | 비고 |
| --- | --- | --- |
| 회원가입 | 완료 | Riot ID, 이름, 비밀번호 가입 |
| 로그인 | 완료 | Riot ID + 비밀번호 |
| 로그아웃 | 완료 | 세션 쿠키 삭제 |
| 비밀번호 재설정 | 부분 | 강사님이 회원 비밀번호 재설정 가능. 사용자의 셀프 비밀번호 찾기는 미완료 |

## 2. 권한

| 권한 | 상태 | 비고 |
| --- | --- | --- |
| 일반 회원 | 부분 | 가입/로그인, 팀 생성/가입/조회 API 있음 |
| 승인된 참가자 | 완료 | 강사님 승인 후 참가자 표시 및 점수제 팀 등록 가능 |
| 팀장 | 부분 | 스크림 팀 생성자가 팀장으로 저장됨. 팀 관리/예약 수정 UI는 부족 |
| 관리자(강사) | 부분 | 장원혁, 서세진 관리자 시드. 회원 승인/비밀번호 재설정 가능 |

## 2-1. 대회 참가 신청

| 항목 | 상태 | 비고 |
| --- | --- | --- |
| 강사님 참가 신청 활성화 | 완료 | 대회별 참가 신청 열기/닫기 가능 |
| 참가 약관 등록 | 완료 | 강사님이 약관 문구 수정 가능 |
| 참가자 약관 동의 후 신청 | 완료 | 승인된 로그인 참가자가 신청 가능 |
| 신청자/미신청자 분리 목록 | 완료 | 강사님 전용 `/participation` 화면과 API |
| 전체 회원 기준 표시 | 완료 | 관리자 제외 전체 회원을 신청/미신청으로 분리 |
