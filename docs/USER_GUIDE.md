# LoL 내전 운영 시스템 사용 설명서

이 문서는 처음 접속한 운영자가 대회를 만들고 마지막 결과까지 진행할 수 있도록 전체 흐름을 설명해.

## 1. 서비스 구성

이 서비스는 두 종류의 대회를 지원해.

| 방식 | 설명 | 핵심 기능 |
|---|---|---|
| 경매 대회 | 팀장이 예산을 사용해 참가자를 낙찰받는 방식 | 팀장 예산, 실시간 입찰, 후보별 타이머, 유찰자 재경매 |
| 점수제 대회 | 참가자 점수 합계 제한 안에서 5인 팀을 구성하는 방식 | 포지션별 점수, 조합 추천, 팀 등록·승인, 토너먼트 |

대회별 참가자, 팀, 경매 기록과 대진표는 서로 섞이지 않아.

## 2. 역할

### 강사님

모든 운영 권한을 가져.

- 대회 생성·선택·삭제
- 강사님 PIN 변경
- 참가자 등록·삭제·점수 수정
- 팀장 지정과 예산·PIN 설정
- 경매 설정 및 진행
- 점수제 제한 설정
- 등록 팀 승인·반려·삭제
- 대진표 생성과 경기 승자 선택

### 팀장

경매 대회에서 본인 팀으로 입장해.

- 강사님이 지정한 팀장과 PIN으로 입장
- 현재 예산 확인
- 숫자로 입찰 금액 직접 입력
- 현재 팀 구성 확인

### 참가자

PIN 없이 공개 화면을 확인해.

- 참가자 소개와 검색
- 점수제 참가자 정보 확인
- 팀 조합 시뮬레이션
- 점수제 팀 등록
- 등록 팀과 대진표 확인
- 경매 진행 관람

## 3. 처음 접속하기

1. 서비스 주소를 열어.
2. 역할 선택 화면에서 원하는 역할을 선택해.
3. 강사님과 팀장은 PIN을 입력해.
4. 참가자는 PIN 없이 입장해.

화면 오른쪽 위의 `역할 전환`을 누르면 로그아웃 후 다른 역할로 다시 입장할 수 있어.

## 4. 강사님 초기 설정

### 4.1 강사님으로 입장

1. 역할에서 `강사님`을 선택해.
2. 강사님 PIN을 입력해.
3. 기본 로컬 PIN은 `1234`이야.

공개 서비스에서는 기본 PIN을 그대로 사용하지 마.

### 4.2 PIN 변경

`강사님 설정 → 강사님 PIN 변경`에서 다음 값을 입력해.

1. 현재 PIN
2. 새 PIN
3. 새 PIN 확인

PIN은 숫자 4~12자리이야. 변경 후 자동 로그아웃되며 새 PIN으로 다시 입장해야 해.

## 5. 대회 관리

### 5.1 새 대회 만들기

`강사님 설정 → 대회 관리`에서:

1. 대회 이름을 입력해.
2. `경매 대회` 또는 `점수제 대회`를 선택해.
3. `대회 생성`을 눌러.

생성한 대회가 즉시 현재 대회로 선택돼.

### 5.2 현재 대회 바꾸기

화면 상단의 `현재 대회` 선택 상자에서 대회를 고릅니다.

대회를 바꾸면 해당 대회의 참가자와 진행 상태만 표시돼.

### 5.3 대회 삭제

대회 카드의 `삭제`를 눌러.

삭제되는 항목:

- 참가자
- 팀장과 예산
- 경매 기록
- 등록 팀
- 대진표와 경기 결과

삭제는 되돌릴 수 없으므로 반드시 대회 이름을 확인해.

## 6. 참가자 등록

참가자 등록은 강사님만 할 수 있으며 경매가 시작되기 전에 완료해야 해.

### 6.1 직접 입력

`강사님 설정 → 참가자 등록 → 직접 입력`에서 다음 값을 입력해.

| 항목 | 설명 |
|---|---|
| 표시 이름 | 화면에 표시할 참가자 이름 |
| Riot ID | 선택 입력. `게임이름#태그` 형식 |
| 티어 | 언랭크부터 챌린저까지 선택 |
| 티어 등급 | 아이언~다이아에서 1~4 드래그바 선택 |
| 주 포지션 점수 | 주 포지션 배치 시 적용할 점수 |
| 부 포지션 점수 | 부 포지션 배치 시 적용할 점수 |
| 주 포지션 | TOP, JUG, MID, ADC, SUP |
| 부 포지션 | 선택 사항 |

부 포지션이 없으면 부 포지션 점수를 입력하지 않아도 돼. 부 포지션을 선택하고 점수를 비워두면 주 포지션 점수를 기본값으로 사용해.

### 6.2 티어 등급 규칙

- 아이언~다이아: I, II, III, IV 등급 사용
- 마스터·그랜드마스터·챌린저: 세부 등급 없음
- 언랭크: 세부 등급 없음

예를 들어 티어 `골드`, 등급 `4`를 선택하면 `GOLD IV`로 저장돼.

### 6.3 Riot API 조회

`Riot API 조회` 탭에서 Riot ID를 입력하면 티어와 프로필 정보를 조회해 등록해.

이 기능은 서버에 유효한 `RIOT_API_KEY`가 설정된 경우에만 작동해.

### 6.4 점수 수정

등록된 참가자 카드에는 주 포지션 점수와 부 포지션 점수 입력칸이 표시돼.

1. 원하는 점수를 수정해.
2. `점수 저장`을 눌러.
3. 해당 참가자가 포함된 점수제 팀의 총점도 다시 계산돼.

### 6.5 참가자 삭제

참가자 카드의 `×`를 눌러.

다음 경우에는 삭제할 수 없어.

- 경매가 이미 시작됨
- 참가자가 점수제 팀에 등록되어 있음

## 7. 참가자 소개와 검색

`선수 소개` 또는 `점수제 참가자` 화면에서 참가자를 확인해.

- TOP부터 포지션 순으로 정렬
- 같은 포지션 안에서는 이름 가나다순 정렬
- 포지션 버튼으로 첫 참가자 이동
- 좌우 버튼 또는 키보드 방향키로 이동
- 이름이나 Riot ID로 검색
- 검색 결과를 클릭하거나 Enter를 눌러 바로 이동

## 8. 점수 계산 규칙

선수 점수는 실제 배치 위치를 기준으로 계산돼.

예:

- 주 포지션 TOP 점수: 10
- 부 포지션 MID 점수: 6

이 선수를 TOP에 배치하면 10점, MID에 배치하면 6점이야.

부 포지션이 아닌 위치에는 배치할 수 없어.

## 9. 점수제 대회 운영

### 9.1 팀 총점 제한 설정

`강사님 설정 → 점수제 팀 총점 설정`에서 제한을 입력하고 저장해.

팀원 5명의 실제 배치 점수 합계가 제한을 넘으면 팀을 등록할 수 없어.

### 9.2 조합 시뮬레이션

참가자는 `조합 시뮬레이션`에서 원하는 선수를 배치할 수 있어.

각 포지션 선택 목록은 다음처럼 구분돼.

- 해당 위치가 주 포지션인 선수: `[주]`
- 해당 위치가 부 포지션인 선수: `[부]`

목록의 점수는 해당 위치에 배치했을 때 실제 적용되는 점수이야.

### 9.3 빈 자리 추천

1. 원하는 선수만 일부 고정해.
2. 나머지는 비워둡니다.
3. `빈 N자리 최적 조합 추천`을 눌러.

추천 기준:

1. 팀 총점 제한에 가장 가까운 조합
2. 부 포지션 배치가 적은 조합
3. 동점이면 목표 점수를 넘지 않는 조합

추천 결과의 `조합 적용`을 누르면 선택 상자에 바로 반영돼.

### 9.4 팀 등록

`팀 등록`에서:

1. 팀명을 입력해.
2. 수정용 PIN을 입력해.
3. TOP/JUG/MID/ADC/SUP 선수를 선택해.
4. 총점을 확인해.
5. `팀 등록 신청`을 눌러.

검사 항목:

- 다섯 포지션 모두 선택
- 같은 선수 중복 사용 금지
- 주/부 포지션 적합성
- 팀 총점 제한
- 이미 다른 팀에 등록된 선수인지 확인

### 9.5 등록 팀 승인

강사님은 `점수제 → 등록 팀`에서 팀을 확인해.

- `승인`: 대진표 생성 대상에 포함
- `반려`: 대진표 대상에서 제외
- `삭제`: 등록 정보 삭제

### 9.6 대진표 생성

승인 팀이 준비되면 `대진표 생성`을 눌러.

- 승인 팀 순서를 무작위로 섞음
- 단판 토너먼트 생성
- 팀 수가 맞지 않으면 부전승 자동 처리
- 대진표 생성 후 팀 등록 종료

### 9.7 경기 결과 입력

강사님이 대진표에서 승리 팀을 누르면:

1. 승자로 표시돼.
2. 다음 라운드에 자동 진출해.
3. 결승 승자를 선택하면 우승 팀이 표시돼.

## 10. 경매 대회 사전 준비

### 10.1 경매 설정

`강사님 설정 → 경매 설정`에서 값을 저장해.

| 항목 | 설명 |
|---|---|
| 방 이름 | 화면 상단에 표시할 경매 이름 |
| 카운트다운 | 후보 한 명의 기본 입찰 시간 |
| 최소 입찰가 | 첫 입찰의 최소 금액 |
| 최소 상승폭 | 이전 최고 입찰보다 올려야 하는 금액 |
| 연장 발동 | 남은 시간이 이 값 이하일 때 입찰하면 연장 |
| 연장 시간 | 막판 입찰 시 추가되는 시간 |

### 10.2 팀장 지정

팀장도 먼저 일반 참가자로 등록해야 해.

`강사님 설정 → 팀장과 예산`에서:

1. 참가자를 선택해.
2. 시작 예산을 입력해.
3. 팀장 입장 PIN을 입력해.
4. `팀장 등록`을 눌러.

팀장은 자신의 주 포지션에 자동 배치되며 비용은 차감되지 않아.

### 10.3 팀장에게 전달할 정보

각 팀장에게 다음 정보를 전달해.

- 서비스 주소
- 본인이 선택해야 할 팀 이름
- 팀장 입장 PIN
- 경매 시작 시각

## 11. 경매 진행

### 11.1 경매장 열기

강사님이 상단의 `경매장`을 누르면:

1. 현재 선택된 대회의 대기 참가자 순서가 무작위로 정해집니다.
2. 첫 후보가 화면에 표시돼.
3. 타이머는 아직 시작되지 않은 대기 상태가 돼.

`강사님 설정`의 `순서를 섞고 경매 시작` 버튼도 같은 동작을 해.

### 11.2 팀장 접속 확인

경매장에는 팀장별 `접속/미접속` 상태가 표시돼.

- 모든 팀장이 접속하기 전: 타이머 시작 버튼 잠김
- 모든 팀장이 접속한 후: `이 후보 타이머 시작` 활성화

팀장은 경매장 화면을 열어둔 상태여야 접속으로 표시돼.

### 11.3 후보별 타이머 시작

강사님이 `이 후보 타이머 시작`을 누르면 현재 후보의 입찰이 시작돼.

후보 한 명의 경매가 끝나면 다음 후보가 표시되지만 타이머는 자동으로 시작되지 않아. 강사님이 다시 팀장 상태를 확인하고 타이머를 눌러야 해.

### 11.4 팀장 입찰

팀장은:

1. `팀장` 역할을 선택해.
2. 자신의 팀을 선택해.
3. PIN을 입력해.
4. 경매장에 입장해.
5. 입찰 금액 입력칸에 숫자를 직접 입력해.
6. `입찰하기`를 눌러.

화면에는 최소 입찰 금액과 남은 예산이 표시돼.

입찰 제한:

- 최소 입찰가 이상
- 현재 최고 입찰 + 최소 상승폭 이상
- 남은 포지션을 채울 최소 예산을 제외한 금액 이하

### 11.5 막판 자동 연장

남은 시간이 `연장 발동` 이하일 때 새 입찰이 들어오면 설정한 `연장 시간`만큼 자동 연장돼.

### 11.6 일시정지와 재개

강사님은 진행 중:

- `일시정지`: 현재 남은 시간을 보존
- `재개`: 보존된 시간부터 다시 시작

### 11.7 낙찰과 유찰

타이머가 0이 되면:

- 최고 입찰이 있으면 해당 팀에 자동 낙찰
- 낙찰 금액만큼 팀장 예산 자동 차감
- 주 포지션이 비어 있으면 주 포지션 배치
- 주 포지션이 찼고 부 포지션이 비어 있으면 부 포지션 배치
- 둘 다 사용할 수 없으면 후보 선수를 벤치에 배치
- 입찰이 없으면 유찰 명단으로 이동

### 11.8 유찰자 재경매

1차 경매가 끝나고 유찰자가 있으면 강사님에게 `유찰자 재경매 시작`이 표시돼.

버튼을 누르면:

1. 유찰자만 새 대기열로 이동해.
2. 경매 라운드가 증가해.
3. 첫 유찰 후보가 대기 상태로 표시돼.
4. 강사님이 후보별 타이머를 다시 시작해.

## 12. 자주 발생하는 문제

### 추천 버튼을 눌러도 동작하지 않음

- 최소 한 명은 선택해야 해.
- 같은 선수를 두 위치에 선택하지 않았는지 확인해.
- 비어 있는 포지션에 배치 가능한 주/부 포지션 선수가 있어야 해.

### 팀 등록 버튼이 비활성화됨

- 다섯 포지션을 모두 선택했는지 확인해.
- 같은 선수가 중복되었는지 확인해.
- 팀 총점 제한을 넘었는지 확인해.

### 부 포지션 점수가 적용되지 않음

- 해당 위치가 선수의 등록된 부 포지션인지 확인해.
- 강사님 설정의 참가자 카드에서 부 점수가 저장되었는지 확인해.

### 경매 타이머 버튼이 비활성화됨

- 모든 팀장이 접속했는지 확인해.
- 현재 후보가 `대기` 상태인지 확인해.
- 팀장이 브라우저에서 경매장 화면을 닫지 않았는지 확인해.

### 팀장이 입찰할 수 없음

- 후보 타이머가 시작되었는지 확인해.
- 올바른 팀과 PIN으로 입장했는지 확인해.
- 최소 입찰 금액과 예산 제한을 확인해.

### 배포 사이트의 데이터가 사라짐

Vercel 서버리스 환경은 로컬 파일을 영구 저장하지 않아. Upstash Redis가 연결되지 않으면 재배포나 인스턴스 교체 시 데이터가 초기화될 수 있어.

## 15. 환경변수

```env
RIOT_API_KEY=
DATA_FILE=/data/state.json
HOST_PIN=1234
SESSION_SECRET=충분히-긴-임의-문자열
UPSTASH_REDIS_REST_URL=
UPSTASH_REDIS_REST_TOKEN=
STATE_REDIS_KEY=lol-auction:state
```

| 변수 | 용도 |
|---|---|
| `RIOT_API_KEY` | Riot ID 조회 |
| `DATA_FILE` | 로컬 JSON 저장 위치 |
| `HOST_PIN` | 최초 강사님 PIN |
| `SESSION_SECRET` | 로그인 쿠키 서명 |
| `UPSTASH_REDIS_REST_URL` | Vercel 영속 저장소 주소 |
| `UPSTASH_REDIS_REST_TOKEN` | Vercel 영속 저장소 인증 |
| `STATE_REDIS_KEY` | Redis에 사용할 상태 키 |

## 18. Vercel 배포

```bash
npx vercel --prod --yes
```

Vercel 배포판은 WebSocket 대신 1초 간격 자동 갱신을 사용해.

영속 운영에는 아래 환경변수가 필요해.

```env
UPSTASH_REDIS_REST_URL=...
UPSTASH_REDIS_REST_TOKEN=...
```

Redis가 연결되지 않으면 화면 상단에 임시 저장 경고가 표시돼.

## 19. 데이터 백업

### 로컬

기본 데이터 파일은 `data/state.json`이야. 서버를 중지하고 파일을 복사하면 백업할 수 있어.

### Vercel

Upstash Redis 콘솔에서 데이터 내보내기 또는 별도 백업 정책을 사용해.

대회 삭제와 더미 데이터 재생성 전에 반드시 백업해.
